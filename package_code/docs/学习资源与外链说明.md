# 学习资源与外链说明

本文档说明：**学习仪表盘 → 知识点速览 → 查看详情 → 资源列表** 中出现的链接为何指向外网、系统如何加载与打开链接，以及如何改为校内或自建资源地址。

---

## 1. 功能路径与界面行为

| 步骤 | 位置 | 操作 |
|------|------|------|
| 1 | 学生端「学习仪表盘」 | 在「知识点速览」区域看到若干知识点卡片（名称、难度、预计时长、标签等）。 |
| 2 | 知识点卡片 | 点击 **「查看详情」**。 |
| 3 | 知识点详情页（路由 `/student/kp/:id`） | 页面展示该知识点的掌握度、先修/后继知识点，以及 **「资源列表」**。 |
| 4 | 资源卡片 | 每条资源显示标题、类型（文档/视频/练习/测验）、难度、预计时长，以及 **完整 URL 文本**；点击 **「开始使用」** 会在 **新浏览器标签页** 中打开该 URL。 |

前端实现要点：

- 「查看详情」由 `KpCard` 触发，跳转到知识点详情路由。
- 资源数据通过接口按知识点 ID 拉取；点击「开始使用」时调用 `window.open(url, "_blank")`，因此链接目标由数据库中的 `url` 字段决定，可以是任意合法 HTTP(S) 地址。

---

## 2. 为何默认是「外网」链接

初始化种子数据脚本会为**每个知识点**自动生成两条示例资源（文档、练习），其 `url` 指向 **哔哩哔哩站内搜索**，形如：

`https://search.bilibili.com/all?keyword=<编码后的关键词>`

关键词由知识点名称与「Python 教程 / Python 练习」等字样拼接后经 URL 编码生成。这样设计的目的通常是：

- **演示完整链路**：无需在本项目内托管大量课件文件，即可展示「资源列表 → 打开学习材料」的交互。
- **快速可运行**：执行 `seed.py` 后学生端即可看到可点的真实链接。

因此，在默认种子数据下，资源列表中的链接**必然依赖公网**；若机器无法访问外网或目标站点策略变更，可能出现打不开或跳转异常等情况。

---

## 3. 数据从哪里来

- 资源存储在数据库表 **Resource** 中，字段包括：`kp_id`（所属知识点）、`type`、`title`、`url`、`difficulty`、`est_minutes` 等。
- 学生端列表接口按 `kp_id` 查询并返回上述字段；前端不做「内网/外网」区分，**原样展示并打开 `url`**。

---

## 4. 如何改为内网地址或自有资源

可按场景选择下列方式之一（或组合使用）。

### 4.1 教师或管理员通过接口维护（推荐用于正式环境）

后端提供资源的增删改接口（需 **teacher** 或 **admin** 角色及对应 JWT）。创建或更新资源时传入自建地址即可，例如：

- 校内课件服务器：`https://intranet.example.edu/course/...`
- 对象存储直链、学校 LMS、Git 文档站等。

更新时修改 `url` 字段后，学生刷新知识点详情页即可看到新链接。

### 4.2 修改种子脚本后重新初始化（仅适用于开发/演示库）

若希望**默认种子**即使用内网或占位地址，可编辑 `backend/seed.py` 中生成 `Resource` 的 `url` 逻辑（当前为拼接哔哩哔哩搜索 URL），改为你方固定模板或配置文件读取。

注意：重新执行 `seed.py` 会向数据库写入数据；若库中已有业务数据，需谨慎处理**重复插入**或**先清空再种子**的策略，避免与生产数据冲突。

### 4.3 直接改数据库

在 `learning_path` 库中更新 `resource` 表的 `url` 列，同样生效；适合批量迁移或一次性修正。

---

## 5. 使用与合规提示

1. **网络环境**：外网链接依赖学生终端与网络策略；内网或离线环境应优先配置内网 `url`。
2. **版权与教学规范**：外链指向第三方平台时，内容版权与平台使用条款由该平台及资源提供方约束；课程建设时建议选用已获授权或自有资源。
3. **安全与隐私**：`window.open` 会跳转到第三方站点，用户应注意钓鱼链接风险；管理端应对录入的 `url` 做基本审核（本项目前端未内置域名白名单）。
4. **体验说明**：种子数据中的链接为**搜索页**而非单一固定课程视频，实际学习内容需用户在搜索结果中自行选择，适合作为演示；正式开课建议改为**精确到具体章节/视频/文档**的 URL。

## 7. 小结

- 资源列表中的链接是否为「外网」**完全由数据库中的 `url` 决定**；默认种子使用哔哩哔哩搜索 URL，因此表现为外网。
- 正式使用时应通过 **教师/管理员接口** 或 **改种子/改库** 将 `url` 换成符合本校网络与版权要求的地址。
- 前端行为是 **新标签页打开链接**，不经过本项目代理或内嵌，故无法单靠前端「关闭外网」；需从数据层替换 URL。

## 8. 站内文档与资源列表入口

项目内提供可在浏览器直接打开的静态说明页（不依赖外网），并已写入种子数据与补种脚本，使其出现在每个知识点的**资源列表**中：

| 项 | 说明 |
|----|------|
| 静态页路径 | `frontend/public/docs/learning-resources-guide.html`，访问 URL 为 **`/docs/learning-resources-guide.html`**（与前端同源，如 `http://127.0.0.1:5173/docs/learning-resources-guide.html`） |
| 资源标题 | `学习资源与外链说明（本项目文档）` |
| 新建库 | 执行 `python seed.py` 时，每个知识点会多出上述一条 `doc` 资源（与哔哩哔哩示例资源并列） |
| 已有库 | 在项目根目录执行：`cd backend && .\.venv\Scripts\python.exe add_guide_resources.py`（幂等，已存在同 URL 则跳过） |

点击资源卡片「开始使用」时，相对路径会在当前前端站点下打开，无需配置外网地址。
